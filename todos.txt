================================================================================
INTUNE ASSIGNMENTS MANAGER — COMPLETE BUILD PLAN
================================================================================

PURPOSE:
A web application for bulk-managing Microsoft Intune app and configuration profile
assignments. The Microsoft Intune admin UI only supports assigning one app at a
time, which is painful at scale. This app lets administrators assign multiple
apps/profiles to multiple groups simultaneously using the MS Graph batch API,
with full support for assignment filters, intent types, and the special
"All Devices" / "All Users" virtual groups.

TARGET USERS:
IT administrators managing Intune environments. Users authenticate with their
own Microsoft Entra ID (Azure AD) account. The app uses delegated permissions,
so users can only do what their Entra roles allow.

DEPLOYMENT:
Hosted on Cloudflare Pages (free plan). No Docker, no self-hosting required.
Users visit the URL, sign in with Microsoft, and start managing assignments.

================================================================================
MICROSOFT GRAPH API REFERENCE
================================================================================

BASE URL: https://graph.microsoft.com/beta

AUTHENTICATION:
- OAuth2 Authorization Code flow with PKCE (handled by MSAL.js in the browser)
- Delegated permissions (user context, not application context)
- Access token passed as: Authorization: Bearer <token>

REQUIRED DELEGATED PERMISSIONS:
- User.Read                                  (sign-in and read user profile)
- DeviceManagementApps.ReadWrite.All         (read/write mobile apps + assignments)
- DeviceManagementConfiguration.ReadWrite.All (read/write config profiles + assignments)
- Group.Read.All                             (list Azure AD groups for assignment targets)

KEY ENDPOINTS:

  List mobile apps:
    GET /deviceAppManagement/mobileApps
    Returns: { value: MobileApp[] }
    Supports $filter, $search, $select, $top, $skip, $orderby, $count

  Get app assignments:
    GET /deviceAppManagement/mobileApps/{id}/assignments
    Returns: { value: MobileAppAssignment[] }

  Apply app assignments (REPLACE all assignments for an app):
    POST /deviceAppManagement/mobileApps/{id}/assign
    Body: { mobileAppAssignments: MobileAppAssignment[] }
    NOTE: This REPLACES all assignments. To add/remove, fetch current first, modify, then POST.

  List configuration policies (Settings Catalog):
    GET /deviceManagement/configurationPolicies
    Returns: { value: DeviceManagementConfigurationPolicy[] }

  Get config policy assignments:
    GET /deviceManagement/configurationPolicies/{id}/assignments
    Returns: { value: DeviceManagementConfigurationPolicyAssignment[] }

  Assign config policy:
    POST /deviceManagement/configurationPolicies/{id}/assign
    Body: { assignments: ConfigurationPolicyAssignment[] }

  List groups:
    GET /groups
    Supports $filter, $search, $top, $select
    NOTE: For $search to work, must include header: ConsistencyLevel: eventual

  List assignment filters:
    GET /deviceManagement/assignmentFilters
    Returns: { value: DeviceAndAppManagementAssignmentFilter[] }

  Batch API:
    POST /$batch
    Body: { requests: [ { id: "1", method: "GET|POST", url: "/path", body: {}, headers: {} } ] }
    Returns: { responses: [ { id: "1", status: 200, body: {} } ] }
    LIMIT: Max 20 requests per batch. For more, send multiple batches sequentially.
    NOTE: Batch requests reference relative URLs (no https://graph.microsoft.com prefix)

ASSIGNMENT DATA MODEL:

  MobileAppAssignment:
    {
      "@odata.type": "#microsoft.graph.mobileAppAssignment",
      "id": "string (read-only, set by Graph)",
      "intent": "required" | "available" | "uninstall" | "availableWithoutEnrollment",
      "target": AssignmentTarget,
      "settings": null | { ... install deadline, restart settings ... }
    }

  AssignmentTarget (one of):
    GroupAssignmentTarget:
      { "@odata.type": "#microsoft.graph.groupAssignmentTarget", "groupId": "guid" }

    AllDevicesAssignmentTarget:
      { "@odata.type": "#microsoft.graph.allDevicesAssignmentTarget" }

    AllLicensedUsersAssignmentTarget:
      { "@odata.type": "#microsoft.graph.allLicensedUsersAssignmentTarget" }

    ExclusionGroupAssignmentTarget:
      { "@odata.type": "#microsoft.graph.exclusionGroupAssignmentTarget", "groupId": "guid" }

  Assignment Filter (applied to any target):
    Add to target object:
      "deviceAndAppManagementAssignmentFilterId": "filter-guid",
      "deviceAndAppManagementAssignmentFilterType": "include" | "exclude"

SPECIAL GROUPS:
- "All Devices" uses AllDevicesAssignmentTarget (no groupId needed)
- "All Users" uses AllLicensedUsersAssignmentTarget (no groupId needed)
- These are NOT real Azure AD groups — they are special target types in the Graph API

RATE LIMITING:
- Graph API returns HTTP 429 with a Retry-After header (seconds)
- Implement exponential backoff: retry after Retry-After value, max 3 retries
- Batch API may return 429 for individual requests within a batch

PROBLEM THIS SOLVES:
